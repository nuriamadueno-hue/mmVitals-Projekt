========================

Important issue found
---------------------
The previous parser selected the root-level XML file `comfig 1.xml` as the config source.
For the current ADC recording this is not the matching capture configuration. The matching
settings are in:

  ADC_Recorded_Data/ADC_Data_LogFile.txt
  ADC_Recorded_Data/ADC_Data_Raw_LogFile.csv

50 ms frame periodicity, 4.8 Msps, and 64.55 MHz/us, while the current capture log/profile

Supported by the revised parser
-------------------------------
- DCA1000 ADC-only .bin files
- IWR1843 / xWR16xx / IWR6843 two-lane DCA1000 format
- xWR12xx / xWR14xx four-lane DCA1000 format
- real-only or complex-IQ ADC formats
- 12-, 14-, or 16-bit ADC output
- 1, 2, or 4 RX for IWR1843/xWR16xx style files
- config from API LogFile.txt, Raw_LogFile.csv, Profile.csv, or explicit XML

Not yet supported / refused with warnings
-----------------------------------------
- TSW1400 captures
- adc_data_RAW_0.bin with UDP packet metadata still present
- advanced-frame multi-subframe separation
- automatic angle-estimation virtual antenna layout
- vital-sign heart/breath extraction; this should be a separate phase-processing script
